### P1 - 高优先级（上架前建议修复）

#### 1. 通知权限被拒绝后无引导 [NotificationManager.swift]
**问题**: 用户拒绝通知权限后，免费版提醒功能无法使用，但App没有提供引导去系统设置开启权限的入口。

**当前行为**:
- SettingsView中开启提醒 → 调用 `requestAuthorization`
- 用户点"不允许" → `granted = false` → 直接关闭提醒
- 用户不知道如何重新开启

**用户体验影响**: ⚠️ 中
- 用户可能误点"不允许"，之后找不到开启方法
- 降低免费版提醒功能的使用率

**修复方案**:
```swift
// NotificationManager.swift 中添加：
func openSettings() {
    if let url = URL(string: UIApplication.openSettingsURLString) {
        UIApplication.shared.open(url)
    }
}

// SettingsView.swift 中，当 authorizationStatus != .authorized 时显示引导：
if appState.reminderEnabled && notificationAuthorizationStatus != .authorized {
    Button(action: { NotificationManager.shared.openSettings() }) {
        Label(L10n.string("去设置中开启通知"), systemImage: "exclamationmark.triangle")
    }
    .foregroundColor(.orange)
}
```

**工作量**: 1小时  
**优先级**: P1 - 建议上架前修复

#### 2. 恢复数据无二次确认 [SettingsView.swift]
**问题**: 点击"恢复数据"选择文件后，直接覆盖现有数据，没有二次确认alert。

**当前代码** (第414-453行):
```swift
private func handleRestoreResult(_ result: Result<[URL], Error>) {
    switch result {
    case .success(let urls):
        guard let url = urls.first else { return }
        // 直接恢复，没有确认alert ⚠️
        ...
    }
}
```

**用户体验影响**: ⚠️ 高
- 误触可能导致数据丢失
- 没有显示将要恢复的数据概要

**修复方案**:
```swift
// 1. 先解析备份文件，获取概要信息
// 2. 显示确认alert：
Alert(
    title: Text(L10n.string("恢复数据")),
    message: Text(String(format: L10n.string("将恢复 %d 条记录、%d 个目标\n此操作会覆盖当前数据，确定继续吗？"), entryCount, goalCount)),
    primaryButton: .destructive(Text(L10n.string("恢复"))) { /* 执行恢复 */ },
    secondaryButton: .cancel()
)
```

**工作量**: 2小时  
**优先级**: P1 - 建议上架前修复

#### 3. CSV导入错误提示不够友好 [BodyEntryStore.swift]
**问题**: 导入失败时，只显示简单错误信息，没有：
- 具体哪行出错了
- 正确的CSV格式示例
- 编码错误的解决建议

**当前代码** (第330-334行):
```swift
if importedCount > 0 {
    return (importedCount, errors.isEmpty ? nil : String(format: L10n.string("%d 行数据跳过"), errors.count))
} else {
    return (0, errors.isEmpty ? L10n.string("未找到有效数据") : errors.first)
}
```

**用户体验影响**: ⚠️ 中
- 用户不知道如何修复CSV文件
- 编码错误时只知道"不支持"，不知道要转成UTF-8

**修复方案**:
1. 在error消息中包含行号：`"第%d行：无法解析日期: %@"`
2. 在importCSV失败时，提供一个"查看正确格式"的按钮，展示标准CSV模板
3. 编码错误时给出具体建议："请使用UTF-8编码保存CSV文件"

**工作量**: 2小时  
**优先级**: P2 - 上架后首次更新修复

#### 6. 没有数据迁移机制
**问题**: 如果未来数据格式变更（如新增字段），旧版本的用户数据可能无法正确迁移。

**用户体验影响**: ⚠️ 低（当前不需要，但未来必要）
- 当前数据格式简单，不太可能变更
- 但最佳实践应该包含版本号和迁移逻辑

**修复方案**:
1. 在 `CodableData` 中添加 `version: Int` 字段
2. 在 `load()` 中检查版本号，执行必要的迁移
3. 当前可以只添加版本号框架，暂时不需要实际迁移逻辑

**工作量**: 2小时  
**优先级**: P3 - 未来版本实现

## 💡 产品优化建议（非技术）

### 提升转化率的建议
1. **Onboarding优化**: 当前3步引导不错，但可以考虑：
   - 增加"跳过"按钮（高级用户可能不喜欢引导）
   - 在最后一步显示"示例数据"，让用户立即看到效果

2. **Paywall优化**: 当前设计不错，但可以：
   - 增加用户评价/评分（如果有）
   - 增加"7天试用"选项（虽然是买断，但可以提供试用）
   - 强调"一次购买，永久使用"的性价比

3. **成就系统优化**:
   - 增加社交分享功能（用户可能愿意分享"连续记录30天"成就）
   - 提供成就提醒（如"再记录1天就能解锁成就"）

### 提升留存率的建议
1. **每周/每月报告**: 自动生成PDF报告，总结用户进步
2. **数据洞察增强**: 当前洞察比较简单，可以增加：
   - "你的体重在下午2点最轻" （时间模式）
   - "你最可能的记录时间是早上8点" （习惯分析）
3. **提醒优化**: 根据用户历史记录时间，智能推荐提醒时间
